CORS là gì trong lập trình Web

CORS (Cross-Origin Resource Sharing) là cơ chế bảo mật của trình duyệt. CORS ngăn website gọi API từ domain khác nếu server không cho phép. Điều này giúp hạn chế các tấn công không mong muốn. Hiểu CORS giúp lập trình viên xử lý lỗi gọi API hiệu quả hơn.

Fetch API trong JavaScript

Fetch API là cách phổ biến để JavaScript gọi API từ trình duyệt. Fetch cho phép: Gửi request GET, POST Nhận dữ liệu JSON từ server Xử lý bất đồng bộ bằng Promise hoặc async/await Fetch API thường được dùng trong các website hiện đại.

Từ Java 11, Java cung cấp HttpClient để gửi HTTP request một cách đơn giản. HttpClient cho phép: Gửi request GET, POST Nhận response dạng String hoặc JSON Giao tiếp với REST API Công cụ này thường được dùng trong các ứng dụng backend Java hoặc khi tích hợp hệ thống.

HTTP cơ bản trong lập trình mạng

HTTP là gì? HTTP (HyperText Transfer Protocol) là giao thức nền tảng của Web, hoạt động theo mô hình Client – Server. Thành phần của HTTP Request (Client → Server) Response (Server → Client) Method: GET, POST, PUT, DELETE Status code: 200, 404, 500… Ví dụ Trình duyệt gửi request GET tới server để lấy trang web. Kết luận Hiểu HTTP là nền tảng quan trọng cho lập trình Web và API.

Lập trình Socket TCP cơ bản với Java

Socket TCP cho phép hai chương trình giao tiếp trực tiếp qua mạng. Trong Java: Server dùng ServerSocket để lắng nghe kết nối Client dùng Socket để kết nối tới server Lập trình socket giúp hiểu rõ cách dữ liệu truyền qua mạng ở mức thấp hơn HTTP.

REST API là gì và vì sao được sử dụng phổ biến

REST API là cách xây dựng dịch vụ web dựa trên giao thức HTTP và các tài nguyên (resource). REST sử dụng các HTTP method để thao tác dữ liệu: GET: lấy dữ liệu POST: thêm dữ liệu PUT: cập nhật dữ liệu DELETE: xoá dữ liệu REST API giúp frontend và backend giao tiếp với nhau rõ ràng, dễ mở rộng và phù hợp cho web, mobile app.
